# ROADMAP — Фазы 10–12 (MOEX ALGO · Историзация для бэктестера · Опционы)

Детальный план-roadmap и TO-DO для фаз 10, 11, 12. Стиль и дисциплина слоёв —
как в `ROADMAP.md`: вся математика в `domain` (без внешних зависимостей и сети),
транспорт/контракты в `data`, персистентность в `storage`, IPC/Tauri в `app`,
UI в `frontend`. Тяжёлые/сетевые части — за cargo-фичами (как `grpc`/`duckdb`/
`tauri`), чтобы кросс-платформенный CI оставался лёгким.

Легенда: `[ ]` — не сделано, `[x]` — сделано, `⏳` — в работе.
Идентификаторы задач (`10.1.2`) стабильны — на них ссылается дизайн-бриф
(`design/claude-design-brief.md`) и трекер.

Принятые решения (из обсуждения):
- Язык: roadmap — RU, дизайн-бриф — EN.
- Модели улыбки опционов (фаза 12): **MOEX-параметрическая + SABR + SVI +
  Каленкович** (≥3 по ТЗ, реализуем 4; выбор модели — в настройках доски).
- LLM-провайдер для итогового резюме (фаза 10): абстракция над провайдерами,
  **по умолчанию OpenRouter**; поддерживаются также Anthropic Claude и OpenAI.

> ⚠️ Точные имена полей и URL ALGOPACK/ISS, а также официальная формула улыбки
> MOEX, требуют авторизованного доступа к документации (login на data.moex.com /
> moexalgo). В задачах ниже эти значения помечены `(verify)` — первый шаг каждого
> релевантного блока: зафиксировать контракт по живому ответу API и закоммитить
> фикстуры в тесты.

## Фаза 11 — Историзация для бэктестера (Finam + MOEX ALGO, выбор источника, локальное хранение)

бэктестера (часть терминала). Источник выбирается в Настройках: **Finam Trade
API** (gRPC bars, уже есть) и **MOEX ALGO** (REST, из фазы 10). Поддержать
наборы тикеров и разные таймфреймы.

- [ ] `11.0.1` Доменная модель историзации (`domain::history`): расширенная

   независимо и может разрабатываться параллельно с данными (12.4).

## Дисциплина и качество (как в проекте)
- Вся математика/правила/промпт-сборка — в `domain`, чисто и в тестах без сети.
- Сетевое/тяжёлое — за фичами (`http`, `moex`, `llm`, `grpc`, `duckdb`, `tauri`);
  кросс-платформенный CI остаётся зелёным без них.
- Секреты — только через резолвер (env/`.env`/keyring), не в localStorage и не в
  логах.
- Каждый PR: `cargo fmt --check`, `cargo clippy -D warnings`, `cargo test
  --workspace`, `npm run build` + `svelte-check`.
- Коммиты — конвенциональные (`feat:`/`fix:`...), как в истории репозитория.
