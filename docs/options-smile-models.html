1. Что такое улыбка и зачем модель

Подразумеваемая волатильность (IV) — это волатильность, которую нужно подставить в формулу ценообразования (Блэка/Башелье), чтобы теоретическая цена совпала с рыночной. Если бы рынок жил по Блэку–Шоулзу, IV была бы одинаковой для всех страйков. На деле IV зависит от страйка: для опционов «вне денег» (особенно путов) она выше — график IV против страйка изгибается вверх по краям. Это и есть улыбка (часто несимметричная — «ухмылка»/skew: путы дороже коллов, т.к. рынок платит за защиту от падения).

Зачем параметрическая модель улыбки:

  • Сглаживание и интерполяция — получить IV для страйков без сделок и непрерывную кривую вместо «зашумлённых» точек.
  • Экстраполяция крыльев — разумная IV далеко вне денег, где котировок нет.
  • Согласованность / отсутствие арбитража — корректные цены и греки по всей доске.
  • Несколько чисел вместо сотен точек — улыбку описывают 3–5 параметров (уровень, наклон/skew, кривизна/«крылья»), удобно хранить, сравнивать во времени и торговать «формой».

Модели ценообразования на MOEX: Black-76 (форвардная версия Блэка, для положительных цен) и Bachelier (нормальная модель, для низких/отрицательных цен). Для маржируемых опционов MOEX безрисковая ставка r = 0. Улыбка задаёт σ(k), а модель ценообразования превращает σ в цену и греки.

3. MOEX — биржевая параметрическая улыбка

MOEX/НКЦ ежедневно строит и публикует параметрическую кривую волатильности по каждой серии опционов — она используется для расчёта теоретической цены, гарантийного обеспечения и риск-параметров. Кривая откалибрована биржей по рыночным котировкам и сглаживает улыбку в устойчивую форму с управляемыми «крыльями».

Идея и форма

Улыбка задаётся как функция стандартизованной моней­ности d = ln(K/F)/(σ₀·√T) — страйк измеряется «в сигмах» (единицах σ·√T). Нормировка на √T даёт кривой срочную структуру: с приближением экспирации |d| растёт, и крылья на фиксированном страйке «поднимаются» (документированное поведение семейства кривых MOEX/НКЦ). Параметры — уровень, наклон (skew) и раздельная кривизна крыльев, с демпфированием (на больших |d| наклон ограничивается, чтобы IV не «улетала»):

d = ln(K/F) / (σ₀·√T)   (стандартизованная моней­ность);
σ(d) = σ₀ + skew·d + c(d)·damp(d)²,   c(d) = c₋ при d<0 (путовое крыло), c₊ при d≥0 (колловое);
damp(d) = tanh(d/wing)·wing   (насыщающее демпфирование крыльев).

Статус верификации. Verified (по методике MOEX/НКЦ, см. §9): ценообразование Black-76/Bachelier, r=0 для маржируемых; нормировка моней­ности на σ·√T и срочная структура («подъём крыльев»); асимметрия крыльев; калибровка биржей по котировкам (взвешенный критерий, вес убывает с удалением от центрального страйка). Unverified (точные коэффициенты биржевой кривой закрыты в деталях методики; первоисточник moex.com/nationalclearingcentre.ru недоступен из окружения): конкретный демпфер крыльев tanh — модельный выбор, сохраняющий документированное качественное поведение (ограниченный, насыщающийся наклон крыльев), а не дословная формула биржи.

ПараметрРоль
σ₀уровень ATM-волатильности (и масштаб нормировки моней­ности)
skewнаклон в σ-единицах (асимметрия путы↔коллы; безразмерный, слабо зависит от T)
c₋, c₊кривизна левого/правого крыла (раздельно — реалистичная асимметрия)
wingмасштаб насыщения крыльев (в единицах σ·√T)
Роль skew в MOEX-улыбке: поворот кривой (путы дороже при skew<0)
Плюсы
  • Рыночный стандарт MOEX: совпадает с расчётом ГО/теорцены биржи — терминал «говорит на языке» клиринга.
  • Устойчивые крылья: пригодна для риск-менеджмента, не даёт абсурдных IV вне денег.
  • Параметры публикуются — можно не калибровать, а брать готовые с доски.
Минусы
  • Точная форма закрыта в деталях методики; вне MOEX модель малоприменима.
  • Сглаживание «под клиринг» может слегка расходиться с лучшими bid/ask маркетмейкера.
  • Нет стохастической динамики (это статическая форма, не модель эволюции волы).

Когда применять: когда нужна согласованность с биржей — теоретическая цена, ГО, риск; как «референс», относительно которого ищут расхождения рынка.

4. SABR (Hagan)

SABR (Stochastic Alpha–Beta–Rho, Hagan и др., 2002) — стохастическая модель: форвард и его волатильность случайны и коррелированы. Её главная ценность — закрытая формула приближения IV (Hagan), ставшая фактическим стандартом на десках по всему миру (в т.ч. азиатских/китайских).

Динамика и формула

dF = α·F^β·dW₁,   dα = ν·α·dW₂,   ⟨dW₁,dW₂⟩ = ρ·dt

Приближение Hagan для лог-нормальной (Black) IV:

z = (ν/α)·(F·K)^((1−β)/2)·ln(F/K)
x(z) = ln[ (√(1−2ρz+z²) + z − ρ) / (1−ρ) ]
σ(K) ≈ α / { (F·K)^((1−β)/2)·[1 + ((1−β)²/24)ln²(F/K) + ((1−β)⁴/1920)ln⁴(F/K)] } · (z/x(z)) · [1 + (…)·T]
ATM: σ_ATM ≈ (α / F^(1−β))·[1 + (…)·T]
ПараметрРольДиапазон
αуровень волатильности (≈ ATM)> 0
β«backbone»/эластичность (0 — нормальная, 1 — лог-нормальная)[0, 1], часто фикс. 0.5
ρкорреляция цена↔вола → наклон/skew(−1, 1)
νвола волатильности → кривизна улыбки> 0
Роль ρ (наклон) и ν (кривизна) в SABR (β=0.5)
Плюсы
  • Финансовый смысл у каждого параметра; β задаёт «скелет», по которому улыбка сдвигается при движении цены — хорошо для хеджирования.
  • Закрытая формула — быстрая калибровка; отраслевой стандарт.
  • Естественно поддерживает нормальный режим (β=0) — удобно с Bachelier-логикой MOEX.
Минусы
  • Приближение Hagan теряет точность на длинных T и далёких крыльях, может давать арбитраж (отрицательные плотности) глубоко вне денег.
  • β и ρ вырождены при калибровке по одной экспирации (β часто фиксируют).
  • Для очень крутых skew/коротких сроков подгонка хуже, чем у SVI.

Когда применять: когда важны динамика и хедж (как улыбка едет за ценой), смешанные положительные/отрицательные форварды, и нужен интуитивный набор параметров. По умолчанию для «китайской школы»/десков — рабочий выбор.

5. SVI (Gatheral)

SVI (Stochastic Volatility Inspired, Gatheral, 2004) параметризует не саму IV, а полную дисперсию w(k) гиперболой. Это лучший «подгоночный» инструмент: точно ложится на рыночные точки и имеет известные условия отсутствия арбитража. Популярна в академической и количественной литературе (в т.ч. китайской — по опционам 50ETF).

Сырая (raw) параметризация

w(k) = a + b·[ ρ·(k − m) + √((k − m)² + σ²) ],   σ_IV(k) = √( w(k) / T )
ПараметрРольОграничение
aобщий уровень дисперсии (вертикальный сдвиг)a + b·σ·√(1−ρ²) ≥ 0
bугол/«раскрытие» крыльев (наклон асимптот)b ≥ 0
ρповорот → асимметрия/skew|ρ| < 1
mгоризонтальный сдвиг (минимум улыбки)—
σсглаживание дна (кривизна у ATM)σ > 0

Условие отсутствия «бабочкового» арбитража (грубо): b·(1+|ρ|) ≤ 4/T; календарный арбитраж исключается монотонностью w(k) по T.

Роль ρ (наклон) и σ (сглаживание дна) в SVI
Плюсы
  • Лучшая подгонка к рыночным точкам по одной экспирации; гибкая форма.
  • Явные условия отсутствия арбитража (бабочка/календарь) — основа «no-arb» поверхности (SSVI/eSSVI).
  • Линейные асимптоты крыльев согласуются с теорией (момент-формула Lee).
Минусы
  • Параметры менее «торговые» (a, m — не прямой уровень/skew); хуже интуиция.
  • Калибровка может попасть в локальный минимум; нужна аккуратная инициализация.
  • Чисто статическая подгонка формы — сама по себе не описывает динамику/хедж.

Когда применять: когда нужна точная бесарбитражная поверхность для переоценки всей доски, генерации непрерывной IV и риск-нейтральной плотности; «золотой стандарт» интерполяции улыбки на одну дату.

6. Каленкович

Алексей Каленкович — российский опционный трейдер; его подход выработан из практики торговли «формой» улыбки. Улыбка параметризуется трейдерскими, наглядными величинами: уровень ATM, наклон (skew), кривизна («куртозис»/крылья) — и обязательно учитывается зависимость от времени (как улыбка уплощается к дальним срокам). Ключевая идея — мерить страйк в стандартизованной моней­ности (в единицах σ·√T), тогда форма улыбки сопоставима между сериями и сроками.

Репрезентативная форма

d = ln(K/F) / ( σ_ATM·√T )   (стандартизованная моней­ность, «сколько сигм до страйка»)
σ(d) = σ_ATM·( 1 + skew·d + kurt·d² ),   с уплощением крыльев и σ_ATM = σ_ATM(T)
ПараметрРоль
σ_ATM(T)уровень при деньгах, зависящий от срока (терм-структура)
skewнаклон в σ-единицах — сопоставим между сериями
kurtкривизна/«толщина хвостов» улыбки
Роль skew и kurt (Каленкович, в единицах σ·√T)
Плюсы
  • Максимально «торговая» интерпретация: уровень / skew / кривизна — то, чем трейдер реально управляет и хеджирует.
  • Стандартизованная моней­ность делает улыбки сравнимыми между сериями/сроками.
  • Явно встроена терм-структура и уплощение со временем — близко к практике.
Минусы
  • Нет единой «официальной» формулы — параметризация по авторским материалам (12.0.5), требует фиксации в коде.
  • Не гарантирует отсутствие арбитража «из коробки» — нужна проверка.
  • Меньше академической базы/инструментов, чем у SVI/SABR.

Когда применять: для трейдерского анализа и торговли формой (skew/кривизна как самостоятельные «активы»), визуального сравнения серий и интуитивной ручной подстройки.

8. Итог и рекомендации

Все четыре модели реализуются в ядре (domain::options::smile), а выбор активной модели — в настройках доски (12.8.1). Рекомендации «когда что»:

Задача / сценарийРекомендуемая модельПочему
Теорцена, ГО, согласие с клирингом MOEXMOEX совпадает с расчётом биржи; готовые параметры с доски
Бесарбитражная переоценка всей доски, непрерывная IV, риск-нейтральная плотность SVIлучшая подгонка + явные no-arb условия

Короткие сроки, очень крутой skewSVI > SABR Hagan теряет точность/устойчивость на крутых крыльях

Практическая связка по умолчанию в терминале: показывать рыночные точки + MOEX (референс биржи) + одну «рабочую» модель (SVI для точной формы или SABR для динамики), с возможностью включить Каленковича для трейдерской интерпретации. Для каждой модели в UI выводить параметры и RMSE подгонки (12.7.3).
